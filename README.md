# 野球プロジェクト

NPB・MLBの配信、試合情報、成績、セイバーメトリクス、契約情報へ素早くアクセスするための個人用野球ポータルです。

## 目的

野球に関する調査、分析、翻訳、コメント添削、データ整理、ブックマーク管理を一元化し、日常的に使える入口として継続改善します。

## 現在のサイト

- 登録サイト: 21件
- お気に入り: MLB.TV、パ・リーグ.com
- 表示分類: 見る、試合・公式情報、選手・成績、高度な分析、契約・移籍、ツール
- リーグ表示: 各カードのMLB・NPBバッジ
- 公開URL: https://mizusan2007.github.io/baseball-bookmarks/
- GitHubリポジトリ: https://github.com/mizusan2007/baseball-bookmarks

### 主な機能

- サイト名、用途、タグを対象にしたリアルタイム検索
- 用途カテゴリとMLB・NPBによる絞り込み
- 利用頻度に応じて大きさを変えたBento Grid
- MLB.TVとパ・リーグ.comの大型ヒーローカード
- 用途別クイックアクセス
- iPhone用ボトムナビゲーション
- PC用左サイドナビゲーション
- 端末内だけに保存する「最近開いたサイト」
- キーボードの `/` で検索、`Escape` で検索・絞り込み解除
- JavaScript無効時の主要リンクと `bookmarks.md` への導線

## ファイル構成

| ファイル | 役割 |
| --- | --- |
| `index.html` | ページ構造、メタ情報、JavaScript無効時の主要リンク |
| `styles.css` | Bento Grid、レスポンシブ表示、配色、ナビゲーション |
| `app.js` | 21件のブックマークデータ、検索、絞り込み、最近開いた履歴 |
| `apple-touch-icon.png` | iPhoneホーム画面およびfavicon用の180×180アイコン |
| `bookmarks.md` | 登録内容を確認するための人間向け一覧 |
| `proposals.md` | 未承認の追加・変更候補。HTMLには未反映 |
| `CHANGELOG.md` | 実施済み変更の履歴 |
| `ideas.md` | 改善案と対応状況 |

外部フレームワークやビルド工程はありません。GitHub Pagesへそのまま配置できます。

## ブックマークの更新方法

実際のサイト表示に使うデータは `app.js` 冒頭の `bookmarks` 配列に集約しています。

1. `app.js` の `bookmarks` 配列へ項目を追加、変更、削除する。
2. `id` は重複しない英数字、`url` はHTTPSを使用する。
3. `category` は既存6分類から選び、`league` は `MLB` または `NPB` とする。
4. `size` は `compact`、`standard`、`wide`、`featured` から選ぶ。
5. `bookmarks.md` を人間向けの確認記録として同期する。
6. `CHANGELOG.md` に変更理由を記録する。
7. 検索、絞り込み、390px・768px・1440pxの表示を確認する。

未承認の候補は `app.js` に追加せず、先に `proposals.md` へ記録します。

## GitHub Pagesへの反映方法

1. このディレクトリの公開ファイルを `baseball-bookmarks` リポジトリのPages公開元へ配置する。
2. `index.html`、`styles.css`、`app.js`、`apple-touch-icon.png` が同じ階層にあることを確認する。
3. 変更をコミットし、Pages対象ブランチへプッシュする。
4. GitHub Pagesの更新後、公開URLでキャッシュを更新して表示とリンクを確認する。

リポジトリ名とPages設定を維持すれば、公開URLは変わりません。

## iPhoneホーム画面への追加方法

1. iPhoneのSafariで公開URLを開く。
2. 共有ボタンをタップする。
3. **ホーム画面に追加** を選ぶ。
4. 表示名「野球ポータル」を確認して追加する。

ホーム画面では `apple-touch-icon.png`、濃紺のテーマカラー、iPhoneのセーフエリア設定が使用されます。

## 運用方針

- 数字、データ、公式サイトなどの一次情報を優先する。
- 事実、解釈、推測を区別する。
- 登録サイトを勝手に削除しない。
- 追加候補は運営状況、無料範囲、既存サイトとの重複を確認する。
- 機能追加より、少ない操作で目的に到達できることを優先する。
